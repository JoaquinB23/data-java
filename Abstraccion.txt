Abstraccion

🔹 ¿Qué es la abstracción?

La abstracción es un principio de la Programación Orientada a Objetos (POO) que consiste en ocultar los detalles internos de implementación y mostrar solo lo esencial y relevante para el usuario de una clase.

👉 En otras palabras:

Te enfocás en qué hace un objeto, no en cómo lo hace.

Sirve para simplificar sistemas complejos y trabajar con ideas más generales.

🔹 ¿Cómo se aplica la abstracción en Java?

En Java, se logra de dos formas principales:

Clases abstractas (abstract)

Pueden tener métodos abstractos (sin código) que las subclases deben implementar.

También pueden tener métodos normales.

Interfaces (interface)

Definen un contrato: qué métodos debe implementar una clase.

No tienen implementación (salvo métodos default y static desde Java 8).

🔹 Ejemplo con clase abstracta
abstract class Figura {
    // Método abstracto
    abstract double area();

    // Método normal
    void mostrarFigura() {
        System.out.println("Soy una figura geométrica.");
    }
}

class Circulo extends Figura {
    double radio;

    Circulo(double radio) {
        this.radio = radio;
    }

    @Override
    double area() {
        return Math.PI * radio * radio;
    }
}

public class Main {
    public static void main(String[] args) {
        Figura f = new Circulo(5);
        f.mostrarFigura(); // Soy una figura geométrica.
        System.out.println("Área: " + f.area()); // Área: 78.54
    }
}


👉 En este caso, Figura representa la abstracción general y Circulo la implementación concreta.

🔹 Ejemplo con interfaz
interface Vehiculo {
    void encender();
    void apagar();
}

class Auto implements Vehiculo {
    @Override
    public void encender() {
        System.out.println("El auto está encendido.");
    }

    @Override
    public void apagar() {
        System.out.println("El auto está apagado.");
    }
}


👉 Acá, la interfaz define el contrato (“todos los vehículos pueden encender y apagar”), pero no dice cómo lo hacen.

🔹 Diferencia clave entre abstracción y encapsulamiento

Abstracción: Se centra en ocultar la implementación y mostrar solo lo esencial. (Qué hace)

Encapsulamiento: Se centra en ocultar los datos internos y protegerlos mediante modificadores de acceso. (Cómo se protege)
